# WEAV-AI 정리/최적화 제안 (실무 기준)

## 원칙
- 기능/경로는 유지하고, 구조만 정리
- 동작에 영향 큰 변경은 사전 합의 후 진행
- 문서/스크립트/인프라/앱 구조를 계층화

---

## 1) 디렉토리 구조 제안 (적용됨)
- `docs/`: 문서 통합 (가이드/체크리스트/설계)
- `ops/`: 운영 스크립트, 배포 설정, launchd/systemd
- `infra/`: 인프라 코드 (docker-compose, nginx, 초기화 스크립트)

적용 내역:
- `infra_WEAV/` → `infra/`
- `infra/ops/` → `ops/`
- 루트의 `*GUIDE.md`, `*CHECKLIST.md` → `docs/`

---

## 2) 프론트 정리 방향
- `components/` 하위 기능별 폴더 정리 (auth/chat/settings)
- 서비스 계층 분리: `services/` → `services/api/`, `services/auth/` 등
- `constants/featureFlags.ts` 기반의 환경별 정책 문서화

---

## 3) 백엔드 정리 방향
- 앱 단위 기능 경계 명확화
  - core: 헬스/공통
  - users: 인증/프로필
  - jobs: 비동기 작업
  - chats: 채팅/폴더
- `weavai/apps/ai` 내부 구조 정리 (router/clients/errors/schemas 명확화)

---

## 4) 환경 변수 정리
- `.env.example` 표준화 (frontend/backend/infra)
- 기능 토글 정책 정리 (ENABLE_EXTERNAL_ACCESS)

---

## 5) 불필요/중복 파일 정리
- 레거시 docker-compose/nginx 파일은 archive 처리
- 중복 문서/가이드 통합
