# WEAV-AI 개발 로드맵 (핵심 흐름 안정화 중심)

## 현재 상태 요약
- 외부 공개/배포는 기본 비활성화 상태
- Google 로그인은 유지
- AI 생성은 백엔드 API 키 기반으로 동작 (개발용)
- 로컬 접근 기준: `http://localhost:8080` (Nginx)
- Jobs 기반 비동기 처리 + 결과 폴링 구조 유지
 - macOS/Windows 모두 Docker 기반 실행 가능

---

## 최근 완료 (2026-01-28)
- AI 폴더 생성 안정화 (플래너 응답 정규화 + 실패 롤백)
- 이미지 확대/다운로드 모달 추가
- 실패 응답 재시도 버튼 추가
- 비디오 모델 엔드포인트 정리 (Sora 2 T2V)

## 1단계: 핵심 흐름 안정화 (최우선)
목표: 목표 입력 → 작업 단계 계획 → 모델 선택 → 생성 → 저장의 E2E 성공률 95% 이상

- 입력/계획/모델 선택 UX 단순화
- 텍스트/이미지/비디오 작업 생성 → 결과 조회(폴링) 안정화
- 저장 경로 정리: Job/Artifact 일관성 + 파일 URL 생성 정상화
- 실패/재시도 정책 점검 (타임아웃, 에러 메시지 일관화)
- 개발용 키 기준 사용량/비용 상한 설정

---

## 2단계: 데이터 안정성 + 운영 기본기
목표: 데이터 유실 및 다운타임 최소화

- 자동 백업/복구 흐름 점검
- 장애 감지/알림 동작 확인
- 비용 폭발 방지: rate limit/쿼터 설계
- 로그 정리 및 에러 추적 기준 확립

---

## 4단계: 외부 공개 재오픈
목표: 외부 유저 대상 오픈

- Cloudflare Tunnel 재활성화
- 도메인 연결/HTTPS 고정
- 관리자/민감 API 접근 제한 및 모니터링 강화

---

## 재오픈 체크리스트 (의미 설명)
"재오픈"은 **외부 유저가 안정적으로 접근 가능하도록 다시 공개**하는 단계.

- [ ] 핵심 흐름 E2E 성공률 95% 이상
- [ ] 장애 알림/백업 정상 작동
- [ ] API 비용 통제(쿼터/limit)
- [ ] 외부 접근 보안(관리자/민감 API 제한)
